Goals of the VLSCI GitHub

To collect together pipelines for bioinformatics (and misc. software tools) To promote collaboration and keeping pipelines up to date.

More generic and user-friendly pipelines will be starred so that they rise to the top.

Guidelines for contributing

New pipelines: To contribute a pipeline, create a new repository and add your code. The name of your repository should include the framework (generally the pipeline tool) and the purpose of the pipeline. We recommend a BSD3 licence, however the choice is up to the creator. The person who first submits the pipeline up takes responsibility for it. For example by responding to pull requests.

Changes to existing pipelines: If you would like to contribute to an existing pipeline, fork it, make your changes and then and submit a pull request.
